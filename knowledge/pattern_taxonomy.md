# Pattern Taxonomy — why X content works

> The **Analyst's reference lens.** Distilled copywriting/growth frameworks, grouped
> by the six dimensions the brief analyzes: **hook · structure · angle · format ·
> timing · pattern.** The Analyst maps each top post onto these so the "why" is
> *transferable* ("uses PAS") instead of vague ("strong hook").
>
> **Rules for the Analyst:**
> 1. Name the *specific* pattern(s), not generic praise.
> 2. If nothing fits, say so — see §0 *No-pattern*. Never force a match.
> 3. Patterns are **hypotheses ranked by frequency**, not proven causes. Confidence
>    rises when a pattern recurs across durable authors (see the pattern library).
> 4. Flag patterns marked ⚠ — they win reach but can be off-brand for a premium agency.
>
> This file is ~2k tokens by design: it fits in one prompt, caches, and stays
> deterministic. (Productionization: swap for a retrieval index once it grows to
> hundreds of full essays.)

---

## 0 · No-pattern (the honesty guard)
Some posts go big for reasons you **cannot replicate**: the author is famous
(authority), it rode a news cycle (timing), it was pure novelty, or it got ratio'd
into reach (controversy). When that's the case, label it `no-replicable-pattern` and
explain the real driver. Do **not** invent a framework.

## 1 · HOOKS (the first line — earns the click/stop)

| Pattern | The tell | Why it works | Example (sample set) |
|---|---|---|---|
| **Contrarian correction** | "X isn't the problem. Y is." Names a belief, negates it. | Pattern-interrupt + curiosity; you must read on to resolve the tension. | Jeevanshu's rant; "Your settings page is a graveyard" |
| **Curiosity gap** | Opens a loop, withholds the payoff ("here's the exact teardown 🧵"). | The brain needs closure → click/expand. | saas_teardowns threads |
| **Big specific number** | A concrete metric up front ("3.1% → 4.6%", "23 clients"). | Specificity = credibility; numbers stop the scroll. | Sid's milestone; checkout teardown |
| **First-person scope** | "I audited 50 B2B SaaS onboardings…" | Authority + implied effort = worth reading. | audit thread (PEAK) |
| **Bold claim / hot take** ⚠ | A flat, debatable assertion ("Figma made designers worse"). | Reach via debate — but often controversy, not value. **Flag it.** | Figma hot-take (guard tripped) |
| **Listicle promise** | "5 empty states done right." | Quantified, skimmable value contract. | "5 done right" thread |
| **Audience question** | "who's the best ___? tag them." | Reply-driven reach; doubles as research/lead-gen. | Sid's question posts (his engine) |
| **"Things I wish I knew"** | Retrospective lessons framing. | Relatable + save-worthy for the earlier-stage reader. | jn "year 1 of product design" |

## 2 · STRUCTURES (how the body is organized)

| Pattern | The tell | Why it works |
|---|---|---|
| **PAS** (Problem→Agitate→Solve) | States a pain, twists the knife, then resolves. | Emotional arc; the solution lands because the pain was felt. |
| **Numbered listicle** | 1/ 2/ 3/ … | Skimmable, promises a finite payoff, easy to save. |
| **Story arc** | Setup → tension → resolution. | Narrative pull; people finish stories. |
| **Before/After + metric** | Old state, new state, the number that moved. | Proof-driven; the metric is the credibility. |
| **Teardown / breakdown** | Dissects a real artifact step by step. | Concrete + educational = high bookmarks. |
| **Comparison** | "$40M ARR SaaS vs a $0 one." | Contrast makes the lesson obvious. |
| **One idea, expanded** | A single insight, no padding. | Respect for attention; punchy, quotable. |

## 3 · ANGLES (the stance)

| Pattern | The tell | Why it works |
|---|---|---|
| **Lesson / insight** | Teaches one transferable thing. | Save-worthy; positions author as a source. |
| **Case study** | Real work + outcome. | Proof beats opinion; sells capability implicitly. |
| **Framework** | A reusable mental model ("config is work, value is the reward"). | Memorable, repeatable, quotable. |
| **Build-in-public** | Behind-the-scenes progress + numbers. | Transparency builds trust + parasocial pull. |
| **Industry callout** ⚠ | Critiques a trend/behavior. | High engagement; keep it principled, not petty. |
| **Hot take / opinion** ⚠ | Pure stance. | Reach via debate — verify it's not just ratio. |

## 4 · FORMATS (the container)

| Pattern | The tell | Why it works |
|---|---|---|
| **Single tweet** | One idea, < 280. | Low friction; great for frameworks/quips. |
| **Thread** | Multi-tweet teardown. | Depth + dwell time; top format for case studies. |
| **Image carousel** | Multiple design shots. | Shows craft; strong for designers. |
| **Video / demo** | Screen-recording of the thing working. | Highest save/share for product reveals. | 
| **Poll / options** | "which do you pick?" + variants. | Cheap to make, reply-driven, audience-flattering. |

> Format note: a bare portfolio image with **no hook** is the *dead-account* trap
> (@bricxlabs). Always wrap craft in a hook, a number, or a human.

## 5 · TIMING

| Pattern | The tell | Why it works |
|---|---|---|
| **Weekday-morning B2B window** | Posted ~Tue–Thu, 9–11am audience time. | Founders/designers scroll before deep work. |
| **Consistency > perfect timing** | Same cadence weekly. | The algorithm and the audience reward reliability; durable beats sporadic-viral. |

## 6 · PATTERN COMBOS THAT REPEAT (highest value)
The engine isn't one trick — it's a *stack*. The recurring winners:
- **Case-study teardown** = first-person scope hook + before/after-metric structure +
  thread format. *(The save-worthy workhorse — model this for Bricx's empty shelf.)*
- **Community question** = audience-question hook + poll/options format. *(Sid's
  proven, cheap, reply-driven engine.)*
- **Milestone proof** = big-number hook + build-in-public angle + video. *(Sid's PEAK.)*
